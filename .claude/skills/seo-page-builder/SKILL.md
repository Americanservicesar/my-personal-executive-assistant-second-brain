## Workflow

### Step 1 — Identify the Page Need

Options:
- **Service+City page:** e.g., "Pressure Washing Conway AR"
- **Blog post:** educational content targeting long-tail keywords
- **Service category page:** overview page for a service line
- **Gap fill:** identify missing pages from the content matrix

### Step 2 — Keyword Research

For each page:
1. Primary keyword: `[service] [city] AR` or `[service] [city] Arkansas`
2. Secondary keywords: variations, related terms, questions people ask
3. Check `memory/marketing.md` → SEO Ranking Changes for current positions
4. Prioritize keywords with: high search volume + low competition + clear commercial intent

### Step 3 — Write the Page

**Page Structure (Service+City):**

```
Title Tag: [Service] in [City], AR | [Brand Name] (under 60 chars)
Meta Description: [Value prop] + [CTA] (under 155 chars)

H1: [Service] in [City], Arkansas

Intro paragraph (100-150 words): What we do, who we serve, why choose us.
Include city name + service naturally.

H2: Our [Service] Services in [City]
- Bullet list of specific services
- Pricing range if applicable

H2: Why Choose [Brand] for [Service]
- Experience, equipment, licensing
- Local presence in [city]

H2: Service Area
- Cities and neighborhoods we cover from [city]
- Embed Google Map if possible

H2: Get a Free Estimate
- CTA: Call [phone] or fill out form
- Link to contact/booking page

H2: Frequently Asked Questions
- 3-5 FAQs targeting long-tail keywords
- Schema markup ready (FAQ structured data)
```

**Blog Post Structure:**
```
Title: [How to / Why / When / X Tips for] [topic] in [city/area]
Meta: [Summary] + [CTA]

H1: [Title]
Intro: Hook + preview of what they'll learn

H2: [Main point 1]
H2: [Main point 2]
H2: [Main point 3]

H2: When to Call a Professional
[Ties back to ASAR services]

CTA: Contact us for [service] in [area]
```

### Step 4 — Internal Linking

Every page should link to:
- Main service category page on the same domain
- 2-3 related service pages
- Contact/booking page
- At least 1 blog post (if exists)

### Step 5 — Publish to WordPress

Use WordPress MCP to:
1. Create the page/post
2. Set title tag and meta description
3. Set featured image (if available)
4. Set categories and tags
5. Publish or save as draft (ask Anthony)

### Step 6 — Track Rankings

After publishing:
1. Log new page to `memory/marketing.md` → SEO Ranking Changes
2. Set baseline: "Published [date], target keyword [X], no rank yet"
3. Check back in 2-4 weeks for initial ranking
4. Update monthly

## Learning Protocol

1. **Before writing:** Check `memory/marketing.md` → SEO Ranking Changes for what's already published and ranking
2. **Before writing:** Check `memory/marketing.md` → Content That Works for high-performing page structures
3. **After publishing:** Log page to `memory/marketing.md` → SEO Ranking Changes with publish date
4. **After 30 days:** Check ranking — update `memory/marketing.md` with position
5. **If page ranks well:** Note the structure, word count, and keyword density in Content That Works
6. **If page doesn't rank:** Note in Content That Flopped — try different structure or keyword angle
7. **Quarterly:** Review full content matrix — identify biggest gaps by search volume
